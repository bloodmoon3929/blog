1주차를 진행하며 가장 신경을 많이 쓴 것은 정석적인 코딩이였다. 그 동안은 개발을 진행할 때, 그저 내키는 대로 개발하며, 문제가 생기면, 처음부터 다시 코딩을 하곤 했다. 

지금까지는 코드의 길이가 짧아 이것이 가능했지만, 앞으로는 더욱 체계적으로 접근할 필요를 느꼈다.

그렇기에 명세, git을 이번 주차의 최고 목표로 잡기로 하였다.

![첨부파일/Pasted image 20251020202427.png](/src/site/img/user/%EC%B2%A8%EB%B6%80%ED%8C%8C%EC%9D%BC/Pasted%20image%2020251020202427.png)
코드는 완전한 MVC 패턴은 아니고, 개발하며 편할 것 같은 방식으로 진행하였다.

model의 Separtor 클래스은 구분자 관련 함수를 내장하고, Enum 형의 ErrorMessage나 Message는 코드에서 출력하는 상수 값들을 저장하였다.

Input과 Output은 말 그대로의 역할을 수행하였고 Validation은 Separator에서 생기는 오류 처리를 해주는 함수로 제작하였다.

```
## 요구사항  
### 기능 요구사항  
- [X] 입력  
> 하나의 문자열을 입력받음 (camp.nextstep.edu.missionutils.Console의 readLine())  
  
- [X] 출력 > "덧셈할 문자열을 입력해 주세요." 문자열 출력  
  
> 결과를 출력함 (결과 : " ")  
  
- [X] 에러처리  
  - [X] 빈 문자열일 때  
  - [X] 구분자만 있고 숫자가 없을 때  
  - [X] 음수의 입력을 받았을 때  
  - [X] "//"으로 시작하나 "\n"이 없는 경우  
  - [X] 구분자 외의 문자가 있을 때,  
> 잘못된 값을 입력시 'IllegalArgumentException' 발생  
  
- [X] 구분자  
  - [X] 기존 구분자  
  - [X] 커스텀 구분자  
> 기본 구분자는 ','와 ':'이며,   
  
> 커스텀 구분자는 문자열의 앞부분에 "//"와 "\n"사이의 문자열을 구분자로 사용함  
  
- [X] 덧셈  
> 구분자로 나눈 문자열을 숫자의 덧셈처럼 더함  
  
- [X] fix 사항  
  - [X] 입력으로 빈 문자열을 받은 경우 0을 반환
```

내가 생각한 오류 사항은 위와 같다.
