# 회고
이번 주차는 코드 리뷰를 하면서 괜찮아 보였던 것 위주로 진행하였다.

우아한 형제들은 Jira를 사용하여 프로젝트를 관리한다 하여 Jira를 사용해 보았다

사용한 후기는 기존에는 README에만 정리하던 것을 온라인에 정리하여, 타임라인을 통해 시간을 관리하니 갈피를 잘 잡을수 있게 된거 같았다.

다만 내 생각에는 Jira가 git로그를 대신하는 기능으로 사용하는 것 같은데 1인 프로젝트에서 굳이 Jira를 사용해야 하는 지에 대해서는 잘 모르겠다.

추가로 Jira에서 commit이나 브랜치 생성등이 가능하다고 하는 데, 나는 사용해 보지 못했다.


두 번째로 mermaid라는 확장에 대해서 알게 되었다. 

1주차에도 코드를 작성할 때, 플로우 차트를 만들어서 로직을 구성했었다.
하지만 이를 종이와 팬을 통해 작성했기에 README에 올릴 정도의 퀄리티가 되지 못하여 올리지 못하였다.

코드 리뷰 중 mermaid를 사용하여 플로우 차트등을 만들어서 README를 꾸민 사람을 보았다.

이 방식이 너무 마음에 들어서 프로젝트 구조, 함수 기능 소개, 다이어그램등을 추가하였다.
